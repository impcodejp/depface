# 画面設計書

## 1. 画面一覧

| # | 画面名 | URL パス | 認証 | コンポーネント |
|---|---|---|---|---|
| 1 | ログイン画面 | `/` | 不要 | `LoginPage.tsx` |
| 2 | メインメニュー画面 | `/menu` | 必須 | `MenuPage.tsx` |
| 3 | ユーザー追加画面 | `/users/add` | 必須 | `UserAddPage.tsx` |
| 4 | 仕様書画面 | `/doc/1` | 不要 | `DocPage.tsx` |

## 2. 共通レイアウト（AppLayout）

認証必須画面（`/menu`・`/users/add`）は `AppLayout` コンポーネントを使用する。

```
┌──────┬──────────────┬────────────────────────────────┐
│ アイ │  サブメニュー │  メインコンテンツ               │
│ コン │  パネル       │                                │
│ バー │  (w-52)       │                                │
│(w-16)│               │                                │
└──────┴──────────────┴────────────────────────────────┘
```

### アイコンサイドバー（左端 64px, bg-slate-900）

| 要素 | 内容 |
|---|---|
| ロゴ | 「df」青い角丸正方形 |
| ホームボタン | `/menu` へ遷移、ホームアイコン＋「ホーム」ラベル |
| カテゴリボタン | 各カテゴリのアイコン＋短縮ラベル（クリックでサブメニューをトグル） |
| ユーザーアバター | ユーザーIDの先頭文字を表示、クリック不可 |
| ログアウトボタン | クリックで `POST /api/auth/logout` を呼び出し、`/` へ遷移 |

アクティブ状態のボタンは `bg-slate-700` になり、左端に青いアクセントバーが表示される。

### サブメニューパネル（最大幅 208px, bg-slate-800）

カテゴリボタン選択時に展開する。カテゴリヘッダー（アイコン＋カテゴリ名）と配下のナビゲーションリンクを表示する。非選択時は幅 0 になりアニメーションで非表示。

### カテゴリ定義

| カテゴリID | ラベル | メニュー項目 | 遷移先 |
|---|---|---|---|
| `users` | ユーザー管理 | ユーザー追加 | `/users/add` |

## 3. 画面詳細

### 3.1 ログイン画面

**URL**: `/`  
**認証**: 不要  
**コンポーネント**: `front/src/pages/LoginPage.tsx`

#### レイアウト

```
┌─────────────────────┬──────────────────────┐
│  左カラム (lg 以上)  │  右カラム（フォーム） │
│  bg-slate-900        │  bg-gray-50           │
│                      │                      │
│  depface             │  ┌────────────────┐  │
│                      │  │ ログイン        │  │
│  業務をシンプルに、   │  │ アカウント情報を │  │
│  チームをスムーズに。 │  │ 入力してください │  │
│                      │  │                │  │
│  社内管理システム     │  │ ユーザーID     │  │
│  depface へようこそ  │  │ [入力欄]       │  │
│                      │  │                │  │
│                      │  │ パスワード     │  │
│                      │  │ [入力欄]       │  │
│                      │  │                │  │
│                      │  │ [エラー表示]   │  │
│                      │  │                │  │
│                      │  │ [ログインBtn]  │  │
│                      │  └────────────────┘  │
└─────────────────────┴──────────────────────┘
```

左カラムはブレークポイント `lg`（1024px）未満で非表示になる。

#### 入力フィールド

| フィールド | type | placeholder | required |
|---|---|---|---|
| ユーザーID | `text` | `user001` | ✓ |
| パスワード | `password` | `••••••••` | ✓ |

#### 状態・ふるまい

| 状態 | 表示内容 |
|---|---|
| 初期 | フォームのみ表示 |
| 送信中 | ボタンが「処理中...」になり disabled |
| 認証エラー | フォーム下に赤色エラーバナーを表示 |
| 認証成功 | `/menu` へ遷移 |

#### API 呼び出し

| タイミング | API | 成功時 | 失敗時 |
|---|---|---|---|
| フォーム送信 | `POST /api/auth/login` | `/menu` へ遷移、AuthContext に token / user_id を保存 | エラーメッセージを表示 |

### 3.2 メインメニュー画面

**URL**: `/menu`  
**認証**: 必須  
**コンポーネント**: `front/src/pages/MenuPage.tsx`  
**レイアウト**: AppLayout

#### メインコンテンツ

```
┌──────────────────────────────────┐
│  （時間帯に応じた挨拶）           │
│  {user_id} さん                  │
│                                  │
│  カテゴリ                        │
│  ┌──────────────┬──────────────┐ │
│  │ ユーザー管理 │              │ │
│  │  ・ユーザー追加             │ │
│  └──────────────┴──────────────┘ │
└──────────────────────────────────┘
```

#### 挨拶ロジック

| 時間帯 | 表示文言 |
|---|---|
| 0〜11時 | おはようございます |
| 12〜17時 | こんにちは |
| 18〜23時 | お疲れ様です |

#### カテゴリカード

各カテゴリをカード形式で表示し、配下のメニュー項目をボタンとして列挙する。クリックで対応するページへ遷移する。

### 3.3 ユーザー追加画面

**URL**: `/users/add`  
**認証**: 必須  
**コンポーネント**: `front/src/pages/UserAddPage.tsx`  
**レイアウト**: AppLayout

#### メインコンテンツ

```
┌──────────────────────────────────┐
│  ユーザー追加                    │
│  新しいユーザーをシステムに登録します│
│                                  │
│  ┌────────────────────────────┐  │
│  │ ユーザーID  [入力欄]        │  │
│  │ ユーザー名  [入力欄]        │  │
│  │ メールアドレス [入力欄]     │  │
│  │ パスワード  [入力欄]        │  │
│  │                            │  │
│  │ [エラーバナー / 成功バナー] │  │
│  │                            │  │
│  │ [ユーザーを追加 Btn]        │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

#### 入力フィールド

| フィールド | type | placeholder | required |
|---|---|---|---|
| ユーザーID | `text` | `user001` | ✓ |
| ユーザー名 | `text` | `山田 太郎` | ✓ |
| メールアドレス | `email` | `example@company.com` | ✓ |
| パスワード | `password` | `8文字以上` | ✓ |

#### 状態・ふるまい

| 状態 | 表示内容 |
|---|---|
| 初期 | フォームのみ表示 |
| 送信中 | ボタンが「処理中...」になり disabled |
| 登録成功 | 緑色バナー「ユーザー「{user_name}」を追加しました」を表示、フォームをリセット |
| 登録失敗 | 赤色エラーバナーを表示 |

#### API 呼び出し

| タイミング | API | 成功時 | 失敗時 |
|---|---|---|---|
| フォーム送信 | `POST /api/users` | 成功バナー表示・フォームリセット | エラーバナー表示 |

## 5. 通知・フィードバック UI

| 種別 | 色 | 使用箇所 |
|---|---|---|
| エラー | 赤（`bg-red-50 border-red-200 text-red-700`） | ログイン失敗、ユーザー登録失敗 |
| 成功 | 緑（`bg-green-50 border-green-200 text-green-700`） | ユーザー登録成功 |
| ローディング | ボタン `disabled` ＋ 「処理中...」テキスト | API 呼び出し中 |
